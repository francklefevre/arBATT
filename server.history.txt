2026-06-02 17:15 — v0.1.3 — Acceleration ("expedite") rule + jsdom test harness.
  Tooling:
    - Added package.json (npm scripts) and installed jsdom (dev dependency) so
      the UI wiring can be tested headlessly. node_modules/ git-ignored.
    - tests/run_all.js : runs every suite, compact PASS/FAIL summary (npm test).
    - tests/test_ui_dom.js : real index.html loaded in jsdom; drives the UI
      (form submit, card taps, mode switch, service chooser, accel panel) and
      asserts the DOM. KEY GOTCHA captured in-file: jsdom fires
      DOMContentLoaded asynchronously, so scripts must be eval'd once and we
      must NOT dispatch DOMContentLoaded ourselves (else bootstrap runs twice,
      binding every handler twice -> one tap = two points). This harness
      retroactively validates the v0.1.1/v0.1.2 UI work end to end.
  Feature - acceleration rule (FFTT manual p.16):
    - scorer.js: new acceleratedBlocks() helper (shared with doubles via the
      api export) -> under acceleration the server changes after EVERY point;
      may start mid-game (records game+total at activation) and PERSISTS to the
      end of the match. TTScorer/DoublesScorer gain activateAcceleration() and
      expose `accelerated` in view(); both derive the cadence from state so
      undo reverts the activation exactly.
    - tests/test_acceleration.js : helper, activate at 0-0, activate mid-game,
      persistence across games, undo revert, doubles rotation every point. Pass.
    - UI: ⏩ Accélération button; once active, an acceleration panel shows the
      receiver return counter (+1 renvoi / reset); reaching ARBATT_ACCEL_RETURNS
      (13) awards the point to the receiver and resets; a normal point also
      resets the counter. Rules screen documents the rule.
  Config:
    - param.json(.example): version 0.1.3; new ARBATT_ACCEL_RETURNS (13).
    - server.py publishes accelReturns in www/app-config.json.
  Verified: `node tests/run_all.js` -> ALL SUITES PASSED (5/5); server.py
    syntax OK; app-config.json serves version 0.1.3 + accelReturns 13;
    server.err.txt empty.
  Deploy impact: SW CACHE_VERSION -> arbatt-v0_1_3 (no new web asset, but
    bumped to refresh caches). package.json/jsdom are dev-only (tests); they do
    not affect the runtime server. .gitignore updated (node_modules/). No
    deploy/ or Docker assets in repo to touch. kurt.json unchanged.

2026-06-02 16:12 — v0.1.0 — Initial increment: project scaffolding + scoring engine.
  Created:
    - server.py            : minimal hermetic static web server for the PWA.
                             Serves a single web-root (www/), rejects path
                             traversal, parameterised host/port/Server header/
                             web-root/default document. Logs via centralised
                             Logger. Generates www/version.json from param.json
                             at startup. Displays version at boot ([BOOT] tag).
    - arbatt_log.py        : centralised logging (single routing function),
                             format "STAMP [TAG] #NNNN message", per-tag enable,
                             error mirror to logs/server.err.txt emptied at boot.
    - arbatt_config.py     : config loader merging config/param.json +
                             config/secret.json, each key overridable by an env
                             var of the same name (typed coercion).
    - config/param.json(.example), config/secret.json(.example).
    - www/ PWA: index.html (menu + setup + scoreboard + over + rules screens),
                css/style.css (mobile-first), js/log.js (client logger),
                js/scorer.js (singles scoring state machine, derived server/ends,
                exact undo), js/app.js (UI controller), manifest.webmanifest,
                sw.js (offline app-shell cache), icons (svg + png 192/512/maskable).
    - tests/test_scorer.js : 21 headless assertions, all passing.
    - README.md, AGENTS_METHODS.md, kurt.json, .gitignore (arBATT section),
      dynamic/ placeholder.
  Verified:
    - node tests/test_scorer.js -> ALL TESTS PASSED (21/21).
    - curl: version.json served, correct MIME types, Server header = "arBATT",
